# 蕾丝图案网站业务流程

## 一、流程总览
设计人员从首页进入工作台后，可浏览本月度值得参考的 Top 10 蕾丝图案，也可以上传客户参考图进行识别匹配。系统会先展示匹配进度，再根据临时规则返回匹配结果。若匹配成功，可进入成衣预览空页面；若无法匹配或确认“不是这一款”，可选择是否创建设计工单，并登记对应客户信息。

## 二、主流程

### 1. 首页浏览
1. 设计人员进入首页。
2. 系统展示 Top 10 蕾丝图案。
3. 系统展示统一更新时间。
4. 设计人员可直接浏览图案，或上传客户参考图开始识图流程。

### 2. 上传识图
1. 设计人员上传客户提供的图案文件。
2. 系统进入匹配中状态。
3. 页面展示匹配进度。
4. 系统通过延时模拟识图过程。
5. 延时结束后执行临时匹配规则。

### 3. 匹配判断
1. 系统读取上传文件的文件名。
2. 系统将文件名与 `pattern` 文件夹中的文件名进行比对。
3. 若文件名一致，返回“匹配成功”。
4. 若文件名不一致，返回“暂未找到您想要的款式”。

### 4. 设计人员确认
1. 若匹配成功，系统展示匹配结果。
2. 设计人员确认是否为客户所需款式。
3. 若确认符合，则显示“点击预览成衣效果”按钮。
4. 若选择“不是这一款”，则进入是否创建设计工单的流程。

### 5. 成衣预览
1. 用户点击“点击预览成衣效果”按钮。
2. 页面跳转到成衣预览页。
3. 当前预览页为占位空页面。
4. 后续再补充正式成衣展示内容。

### 6. 设计工单与客户登记
1. 若系统无法匹配或设计人员选择“不是这一款”，系统询问是否创建设计工单。
2. 若确认创建，则进入客户信息登记表单。
3. 设计人员填写客户姓名与客户联系方式。
4. 设计人员点击“登记并创建工单”。
5. 系统生成唯一工单编号，将状态设为待处理并返回成功提示。

## 三、分支流程

### 1. 匹配成功分支
- 用户上传图案。
- 系统模拟匹配进度。
- 文件名比对成功。
- 用户确认符合。
- 显示成衣预览按钮。
- 跳转到预览空页面。

### 2. 匹配失败分支
- 用户上传图案。
- 系统模拟匹配进度。
- 文件名比对失败。
- 提示“暂未找到您想要的款式”。
- 询问是否创建设计工单。
- 若确认创建，则登记客户姓名和联系方式并生成工单编号。

## 四、页面流转
1. 首页
2. 上传识图页
3. 匹配进度页
4. 匹配结果页
5. 成衣预览空页面
6. 设计工单与客户登记页

## 五、状态说明

### 1. 初始状态
- 设计人员未上传客户参考图。
- 页面展示 Top 10 图案与上传入口。

### 2. 匹配中状态
- 设计人员已上传客户参考图。
- 系统正在模拟识图过程。
- 页面显示进度信息。

### 3. 匹配成功状态
- 系统完成文件名匹配。
- 页面提示匹配成功。
- 设计人员可以确认并预览。

### 4. 匹配失败状态
- 系统完成文件名匹配但未命中。
- 页面提示未找到目标款式。
- 设计人员可以选择是否创建设计工单。

### 5. 工单创建完成状态
- 设计人员已登记客户姓名与联系方式。
- 系统反馈工单编号及创建成功状态。

## 六、流程图文字版
```text
首页浏览
  -> 上传图案
  -> 显示匹配进度
  -> 文件名匹配判断
      -> 匹配成功
          -> 用户确认符合
              -> 进入成衣预览空页面
      -> 匹配失败
          -> 提示未找到款式
          -> 是否创建设计工单
              -> 是
                  -> 登记客户姓名与联系方式
                  -> 创建工单并返回编号
              -> 否
                  -> 结束流程
```

## 七、补充说明
1. 当前匹配规则只是临时方案，重点是先跑通业务闭环。
2. 匹配进度用延时模拟，后续可替换为真实识别状态。
3. 成衣预览页当前仅做占位，避免影响整体流程交付。
